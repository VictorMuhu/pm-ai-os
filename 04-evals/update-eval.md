# Update Eval

> Evaluation rubric for AI-generated stakeholder updates or exec briefs.

## Dimensions

| Dimension | Score (1–5) | Notes |
|---|---|---|
| Lead with status | | Is the status/headline visible in the first 3 lines? |
| Progress clarity | | Does it report what changed, not what's in-progress? |
| Decision documentation | | Are decisions logged with rationale? |
| Risk honesty | | Are risks surfaced with owners, not sanitized? |
| Ask specificity | | Are asks explicit and actionable? |
| Brevity | | Is it readable in under 2 minutes? |

## Readiness rating

- **Ready**: All dimensions 4+
- **Needs revision**: Missing asks, buried status, or missing risk owners

## Common failure modes

- Status buried after multiple paragraphs of context
- Risks listed without owners or mitigations
- Asks implied rather than stated
- Reports activity ("we worked on…") instead of progress ("we shipped / decided / unblocked…")
