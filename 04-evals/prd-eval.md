# PRD Eval

> Evaluation rubric for AI-generated or human-drafted PRDs.

## Dimensions

| Dimension | Score (1–5) | Notes |
|---|---|---|
| Problem clarity | | Is the problem customer-centric and specific? |
| Evidence quality | | Are claims grounded in real evidence? |
| Metric quality | | Are success metrics measurable and owned? |
| Requirement completeness | | Are requirements unambiguous and implementable? |
| Edge case coverage | | Are edge cases addressed or explicitly deferred? |
| Out-of-scope clarity | | Is out-of-scope stated explicitly? |
| Open question hygiene | | Are open questions flagged with owners? |

## Readiness rating

- **Ready**: All dimensions 4+, no critical gaps
- **Needs minor revisions**: 1–2 dimensions below 4, no blockers
- **Needs major revisions**: 3+ dimensions below 3, or any blocker present

## Common failure modes

- Problem statement contains solution language
- Metrics are outputs (e.g., "launch the feature") not outcomes
- Edge cases are silently ignored
- Open questions are buried or missing owners
